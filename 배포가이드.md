# 블로그 도구 배포 가이드
# 한 번만 하면 평생 무료로 누구나 사용 가능!

---

## 📁 파일 구조
```
blog-nextjs/
├── pages/
│   ├── index.jsx          ← 메인 블로그 도구
│   └── api/
│       └── keyword-stats.js  ← 네이버 API 서버
├── package.json
├── next.config.js
├── .env.local             ← API 키 (GitHub에 올리면 안 됨!)
└── .gitignore
```

---

## STEP 1. GitHub 가입 및 저장소 만들기

1. https://github.com 접속 → [Sign up] 클릭
2. 이메일/비밀번호 입력 후 가입
3. 로그인 후 우측 상단 [+] → [New repository] 클릭
4. Repository name: `blog-tools` 입력
5. Public 선택 → [Create repository] 클릭
6. 화면에 나오는 주소 복사 (예: https://github.com/내아이디/blog-tools.git)

---

## STEP 2. 파일 올리기 (GitHub Desktop 사용 - 가장 쉬운 방법)

1. https://desktop.github.com 접속 → GitHub Desktop 설치
2. File → Add Local Repository → blog-nextjs 폴더 선택
   (없으면 File → New Repository → blog-nextjs 폴더 지정)
3. 왼쪽에 파일 목록 보이면 하단 [Commit to main] 클릭
4. 상단 [Publish repository] 클릭
   → Private 체크 해제 (Public으로)
   → [Publish repository] 클릭

---

## STEP 3. Vercel 가입 및 배포

1. https://vercel.com 접속 → [Sign Up] → [Continue with GitHub] 클릭
2. GitHub 계정으로 로그인 허용
3. [Add New Project] 클릭
4. blog-tools 저장소 찾아서 [Import] 클릭
5. 그냥 [Deploy] 클릭 (설정 건드리지 않아도 됨)
6. 배포 완료! 🎉 → `https://blog-tools-내아이디.vercel.app` 주소 생성

---

## STEP 4. 네이버 API 키 환경변수 설정 (중요!)

GitHub에는 .env.local을 올리면 안 되므로 Vercel에 직접 입력해야 합니다.

1. Vercel 대시보드 → blog-tools 프로젝트 클릭
2. 상단 [Settings] 탭 클릭
3. 왼쪽 [Environment Variables] 클릭
4. 아래 3개 입력:

   | Name                | Value |
   |---------------------|-------|
   | NAVER_API_KEY       | 0100000000e7a42a13416cd96be8f652ec09de235096af9030b87c2fc59b3a2043d7480c86 |
   | NAVER_SECRET_KEY    | AQAAAADnpCoTQWzZa+j2UuwJ3iNQEjNS5DgYdZ+bvpeQTo17pA== |
   | NAVER_CUSTOMER_ID   | 545035 |

5. 각 항목 입력 후 [Save] 클릭
6. 상단 [Deployments] → 최신 항목 우측 [...] → [Redeploy] 클릭
